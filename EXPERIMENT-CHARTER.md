# EXPERIMENT CHARTER — 8주 검증 실험 헌법

> **이 문서는 사장님(사용자)의 명시적 승인 없이 수정 금지.** AI는 이 문서를 고칠 수 없다.
> 계획과 다르게 행동해야 한다면, 먼저 `strategy-board.json`의 `deviations`에 자백 기록 후 사장님 승인을 받는다.
> 제정: 2026-07-10 · 개정 이력: (없음)

---

## 1. 실험 정의

- **가설**: "색인되는 표면(티스토리)에 결정 콘텐츠(비교표+판단기준+제휴)를 올리면, 8주 내 색인→유입→제휴 클릭이 발생한다."
- **이 실험의 진짜 목적**: 블로그 수익 자체가 아니라, **"자율 에이전트가 병목을 진단하고 전략을 실행해 지표를 움직인다"는 증거**를 만드는 것 (프로젝트 목표 = B: 자비스).
- **예산 상한**: ₩30,000 (초과 지출 절대 금지)
- **발행 상한**: 주당 신규 글 3편 이하 (양산 금지). 모든 발행 글은 CLAUDE.md 팩트 무결성 규칙 통과 필수.
- **기존 자산**: vercel 26편은 크롤 0(구글이 모름) → 중복 페널티 없이 티스토리 이식 가능. 이식 시 원본으로 취급됨.

## 2. 킬라인 (숫자 — 합리화 불가)

| ID | 시점 | 조건 | 조치 |
|----|------|------|------|
| **K1** | W4 종료(8/9) | 티스토리 발행 글의 검색 색인(다음 또는 구글) **0건** | 표면 교체 **1회만** 허용. 교체 후 W6 종료(8/23)까지도 색인 0 → **실험 중단** |
| **K2** | W8 종료(9/6) | 제휴 링크 클릭 누적 **0건** | **실험 중단**, 회고 작성, 배운 것만 남김 |
| **K3** | 상시 | 예산 ₩30,000 초과 필요 | 지출 금지, 사장님 결정으로만 해제 |

킬라인 도래 시 AI는 "조금만 더"를 제안할 수 없다. 계속하려면 사장님이 이 헌법을 개정해야 한다.

## 3. 8주 스케줄 (주차는 날짜로 판정 — 오늘 날짜로 계산)

| 주차 | 기간 | 임무 |
|------|------|------|
| **W0** | 7/10–7/12 | 셋업: ①티스토리 계정 확정(사장님) ②제휴 가능 니치 3개 웹 리서치(AI 툴 제휴 프로그램: 수수료율·가입조건) ③기존 26편 중 결정형 글 5편 선별 + 팩트 재검증 |

| **W2** | 7/20–7/26 | 색인 1차 측정(다음/구글 site:). 제휴 프로그램 가입 신청 |

밀린 주가 있으면(트리거 누락) 해당 주 임무를 통합 수행하되, 킬라인 날짜는 **연장되지 않는다**.

## 4. "주간 사이클" 실행 절차 (매번 이 순서 그대로 — 재량 금지)

1. **로드**: 이 문서 전문 + `src/memory/strategy-board.json` + `PROGRESS.md` 최근 세션 로그
2. **주차 판정**: 오늘 날짜를 §3 표에 대조 → 이번 주차 임무 확정
3. **측정** (방법 고정):
   - 다음 색인: 다음 검색 `site:{티스토리주소}` (WebFetch)
   - 구글 색인: `site:` 검색 + 필요시 GSC (브라우저 익스텐션, 안 되면 사장님께 스크린샷 요청)
   - 티스토리 유입: 티스토리 관리자 통계 (스크린샷 요청)
   - 제휴 클릭: 제휴 플랫폼 대시보드 (스크린샷 요청)
   - **모든 숫자에 출처 명기. 검증 불가 수치는 "미확인"으로 적는다. 날조 금지.**
4. **기록**: `strategy-board.json` funnel.snapshots에 append + 킬라인 상태 갱신
5. **실행**: 이번 주차 임무 수행. 단, **공개 게시·지출·계정 생성·비가역 행위는 실행 직전 사장님 승인** (예/아니오로 답할 수 있게 물을 것)
6. **로그**: `PROGRESS.md`에 세션 기록 append
7. **리포트**: 아래 §5 형식 그대로 제출

## 5. 주간 리포트 고정 형식

```
## 주간 리포트 W{n} ({날짜})
1. 지표: [티스토리] 발행 n / 색인 n / 유입 n / 제휴클릭 n  [vercel 대조군] 크롤 n / 색인 n
2. 킬라인: K1 {상태} · K2 {상태} · 예산 사용 ₩{n}/30,000
3. 이번 주 한 일: (3줄 이내)
4. 다음 주 임무: (헌법 §3 기준)
5. 승인 필요: (없으면 "없음")
6. 이탈(deviation): (없으면 "없음")
```

## 6. 역할 분담

- **사장님**: 주 1회 "주간 사이클" 트리거 (캘린더 반복 알림 권장) · 승인 요청에 예/아니오 · 킬라인 도래 시 최종 결정
- **AI**: 그 외 전부 — 측정, 판단, 실행, 기록, 리포트. 단 §4의 승인 규칙과 킬라인은 절대 우회 불가.

## 7. 금지 목록 (이 실험 기간 중)

- vercel에 신규 글 발행 (vercel은 대조군으로 동결)
- 수동 GSC 색인 요청 재시도 (quota 무력 — 실측 완료)
- 쓰레드 백링크를 크롤 목적으로 재실행 (무력 — 실측 완료)
- 측정된 개선 없이 같은 레버 재실행
- 웹 리서치 없이 과거 지식만으로 전략 확정
- 킬라인 날짜/조건의 재해석
