## 핵심 요약

- 자연어로 앱을 만드는 "바이브코딩" 도구 3종 — **v0은 UI/프론트엔드 특화, Bolt.new은 풀스택 속도, Replit은 자율 개발 에이전트**로 용도가 다르다
- 2026년 7월 기준 유료 진입 가격: **Bolt.new Pro $25/월, Replit Core $25/월, v0 Team $30/월(인당)** — v0의 $20 개인 플랜(Premium)은 신규 가입이 막힌 상태
- 세 도구 모두 프로토타입·MVP까지가 강점 구간 — 인증·결제·성능 최적화가 걸린 프로덕션은 여전히 사람 개발자의 검증이 필요하다

## 자연어 프로그래밍, 어디까지 왔나

가트너는 2022년 전망에서 로우코드 개발 기술 시장이 2026년 445억 달러(약 60조 원) 규모에 이를 것으로 예측했다(Gartner, 2022년 12월 전망). 자연어로 코드를 생성하는 도구들이 이 시장의 핵심 동력이 되고 있고, 최근에는 "바이브코딩(vibe coding)"이라는 이름으로 비개발자층까지 사용자가 넓어졌다.

다만 "코드 없이 앱을 만든다"는 말은 과장이다. 정확히는 **"코드를 직접 작성하는 비율을 줄인다"**에 가깝다. 이 글은 그 대표 도구 3종 — Vercel v0, Bolt.new, Replit Agent — 를 가격·출력 품질·배포·기술 스택 기준으로 비교한다.

## 도구별 상세 분석

### Vercel v0 — 디자이너와 프론트엔드 개발자용

**잘하는 것**:
- 프롬프트 → UI 컴포넌트 변환 품질이 세 도구 중 가장 높음
- shadcn/ui 기반으로 일관된 디자인 시스템 출력
- Vercel 배포 파이프라인과 자연스러운 연결, 무료 플랜부터 GitHub 동기화 지원

**한계**:
- 복잡한 백엔드 로직에 약함
- Vue·Svelte·플레인 HTML 코드도 생성은 가능하지만, 실시간 프리뷰와 출력 품질은 React/Next.js에 최적화돼 있음
- 대규모 앱보다는 컴포넌트/페이지 단위에 적합

**적합한 사용자**: 디자인 → 코드 변환이 필요한 디자이너, React 기반 프론트엔드를 빠르게 만들 때

### Bolt.new — 스타트업 창업자용

**잘하는 것**:
- 풀스택 앱을 가장 빠르게 생성
- 프레임워크 선택의 폭이 넓음
- 실시간 미리보기로 즉각적인 피드백
- Pro 플랜은 미사용 토큰 이월(rollover) 지원 — 간헐적 사용자에게 유리

**한계**:
- UI 디테일은 v0보다 떨어짐
- 복잡한 상태 관리에서 버그 빈도 높음
- 생성된 코드의 구조가 일관적이지 않을 때 있음

**적합한 사용자**: MVP를 빠르게 만들어 검증하려는 창업자, 비개발자 중 기술 이해도가 있는 사람

### Replit Agent — 개발자 프로토타이핑용

**잘하는 것**:
- 에이전트가 스스로 계획 → 실행 → 디버깅까지 수행 (Core 플랜은 에이전트 2개 병렬 실행)
- 터미널 접근, 패키지 설치 등 개발 환경 전체 제어
- 복잡한 로직과 백엔드 작업에 가장 유능

**한계**:
- UI 품질은 세 도구 중 가장 낮음
- 에이전트 실행 시간이 길어질 수 있고, 크레딧 소모가 작업 복잡도에 따라 예측하기 어려움
- 호스팅이 Replit 환경에 종속

**적합한 사용자**: 빠르게 프로토타입 만드는 개발자, 백엔드 중심 프로젝트, 학습 목적

## 선택 가이드

| 상황 | 추천 도구 | 이유 |
|------|----------|------|
| 랜딩 페이지 빠르게 만들기 | v0 | UI 품질 최고 |
| MVP 전체를 하루 만에 | Bolt.new | 풀스택 속도 최고 |
| 복잡한 로직이 있는 프로토타입 | Replit Agent | 에이전트 자율성 |
| 디자인 시안 → 코드 변환 | v0 | 컴포넌트 정밀도 |
| 비개발자의 첫 앱 | Bolt.new | 진입장벽 가장 낮음 |
| 학습/실험 용도 | Replit Agent | 전체 과정 관찰 가능 |
| 혼자 쓰는데 예산 최소화 | Bolt.new 또는 Replit | 개인 유료 $25 — v0은 개인 플랜 신규 가입 중단으로 Team $30부터 |

## 언제 여전히 개발자가 필요한가

세 도구 모두 공통적으로 한계가 있는 영역:

| 영역 | 한계 내용 |
|------|----------|
| 인증/보안 | OAuth, JWT 등 복잡한 인증 플로우 — 생성 코드에 보안 허점 빈번 |
| 결제 연동 | Stripe, 토스페이먼츠 등 실 결제 — 수동 검증 필수 |
| 성능 최적화 | 대규모 데이터, 동시 접속 처리 — AI 생성 코드는 최적화 고려 부족 |
| 접근성 | WCAG 준수 — 자동 생성 코드의 접근성은 불완전 |
| 유지보수 | 6개월 후 코드 수정 — 구조를 이해하는 사람이 필요 |
| 규제 준수 | 개인정보보호법, 의료/금융 규제 — 도메인 전문 지식 필요 |

**결론**: 이 도구들은 **"개발자를 대체"하는 게 아니라 "개발 시작점을 바꾸는"** 것이다. 0에서 60%까지의 속도를 크게 높여주지만, 나머지 40%는 여전히 사람의 영역이다.

## 비용 비교 (2026년 7월 기준, 월 결제)

각 사 공식 pricing 페이지에서 확인한 현행 요금이다. 셋 다 크레딧/토큰 기반 과금이라 "무제한"은 없다는 점에 주의.

| 플랜 | v0 | Bolt.new | Replit |
|------|---|---|---|
| 무료 | Free — 월 $5 크레딧, 일 7메시지 | Free — 일 30만·월 100만 토큰 | Starter — 일일 에이전트 크레딧, 앱 게시 1개 |
| 개인 유료 | Team $30/월(인당)* | Pro $25/월 (토큰 1,000만~, 이월 가능) | Core $25/월 ($25 크레딧, 연간 결제 시 $20) |
| 상위 | Business $100/월(인당) | Teams $30/월(인당) | Pro $100/월 ($100 크레딧, 협업 15인) |
| 엔터프라이즈 | 문의 | 문의 | 문의 |

\* v0의 $20 개인 플랜(Premium)은 2026년 상반기에 공식 가격 페이지에서 내려갔고 신규 가입이 막힌 상태다(기존 가입자만 유지). 현재 신규 사용자의 유료 진입점은 Team $30부터다.

세 도구 모두 무료로 체험은 가능하지만, 실제 프로젝트 하나를 완성하려면 유료 플랜이 거의 필수다. 월 $25~30 선에서 시작할 수 있으니, 무료 티어로 같은 프롬프트를 세 도구에 던져보고 출력 스타일이 맞는 쪽에 결제하는 순서가 합리적이다.

## 만든 앱을 실서비스로 배포하려면

프로토타입까지는 세 도구의 내장 호스팅으로 충분하다. 문제는 그다음이다 — 커스텀 도메인, 상시 가동 백엔드, 데이터베이스가 붙는 순간 각 플랫폼의 종속과 과금 구조가 발목을 잡는다.

- **v0**: Vercel 배포가 기본이라 프론트엔드는 매끄럽지만, 상시 실행되는 백엔드 서버·워커가 필요하면 서버리스 함수 제약에 부딪힌다
- **Bolt.new**: Pro부터 커스텀 도메인 호스팅을 지원하지만, 앱이 커지면 코드를 내려받아 별도 인프라로 옮기는 경우가 많다
- **Replit**: 배포까지 한 플랫폼에서 되는 대신, 트래픽이 늘면 크레딧 소모가 커지고 Replit 환경 밖으로 나가기 어려워진다

이때 흔한 탈출 경로가 **생성된 코드를 GitHub으로 빼서 범용 배포 플랫폼에 올리는 것**이다. Railway 같은 서비스는 GitHub 저장소를 연결하면 백엔드·DB·크론잡까지 사용량 기반으로 배포할 수 있어, 노코드 빌더에서 만든 앱을 "진짜 서비스"로 승격시키는 다음 단계로 자주 쓰인다.

## 정리

- **UI 중심, React 프로젝트** → v0
- **빠른 풀스택 MVP, 개인 예산 $25** → Bolt.new
- **복잡한 로직, 에이전트 기반 개발** → Replit Agent
- **프로덕션 배포** → 세 도구 모두 추가 작업 필요. 코드 소유권(GitHub 내보내기)과 외부 배포 경로를 미리 계획할 것

가격과 플랜 구성은 이 시장에서 분기 단위로 바뀐다. 결제 전 반드시 각 사 공식 pricing 페이지에서 최신 요금을 확인하길 권한다. (본문 가격: 2026년 7월 기준)
