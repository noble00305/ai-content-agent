## 핵심 요약

- VRAM 8GB면 Qwen3-4B·Phi-4-mini, 16GB면 Qwen3-14B, 24GB면 Qwen3-32B·Qwen3-30B-A3B까지 단일 GPU로 실행 가능
- MoE 모델은 "활성 파라미터"가 아니라 **총 파라미터를 전부 메모리에 적재**해야 함 — DeepSeek V3.2는 Q4 양자화로도 약 400GB라 개인 PC에서는 사실상 불가, API 이용이 현실적
- 상업적 이용이 자유로운 라이선스는 Qwen3(Apache 2.0), Phi-4-mini(MIT), DeepSeek V3.2(MIT). Llama 4만 조건부

이 글은 AI가 공개 자료(Hugging Face 모델 카드, 각 사 공식 문서)를 대조해 정리한 큐레이션입니다. 파일 크기 등 수치는 2026년 7월 기준 출처를 명시했습니다.

## 왜 지금 로컬 LLM인가

클라우드 API 비용이 프로젝트 규모에 따라 월 수십~수백 달러까지 올라가는 상황에서, 로컬 LLM은 전기료 외 추가 비용이 없다는 장점이 있다. 개인정보를 외부로 보내지 않아야 하는 업무, 오프라인 환경, 커스텀 파인튜닝처럼 로컬 실행이 필수인 시나리오도 늘고 있다.

다만 로컬 LLM 입문에서 가장 흔한 실수가 하나 있다. **MoE(Mixture of Experts) 모델의 "활성 파라미터" 숫자만 보고 내 GPU에서 돌아갈 거라 착각하는 것.** MoE는 추론 시 일부 전문가만 계산에 쓰지만, 어떤 전문가가 호출될지 모르기 때문에 **전체 파라미터를 전부 메모리에 올려야 한다**. 예컨대 DeepSeek V3.2는 활성 파라미터가 37B지만 총 685B라서, 4비트 양자화를 해도 약 400GB급 메모리가 필요하다. 이 글의 표는 전부 "총 파라미터 적재량" 기준으로 작성했다.

## 모델별 상세 비교

### Qwen3 시리즈 (Alibaba Cloud)

- **파라미터**: 0.6B / 1.7B / 4B / 8B / 14B / 32B (dense) + 30B-A3B / 235B-A22B (MoE)
- **라이선스**: Apache 2.0 (상업적 사용 자유)
- **특징**: dense부터 MoE까지 라인업이 촘촘해 VRAM 예산에 맞춰 고르기 가장 쉬움. 30B-A3B는 활성 3B로 계산량은 적지만, 적재에는 Q4 기준 18.6GB가 필요(Unsloth GGUF 기준)
- **한국어**: 다국어 학습 비중이 높아 한국어 생성 품질에 대한 커뮤니티 평가가 좋은 편
- **코딩**: 공개 코딩 벤치마크(LiveCodeBench 등)에서 오픈소스 상위권

### Llama 4 시리즈 (Meta)

- **파라미터**: Scout (총 109B, 16전문가, 활성 17B) / Maverick (총 400B, 128전문가, 활성 17B)
- **라이선스**: Llama Community License (월 활성 사용자 7억 이하 무료, 초과 시 별도 협의)
- **특징**: Scout는 Meta 발표 기준 최대 1,000만 토큰 컨텍스트. 단, 총 109B라 Q4_K_M GGUF가 65.4GB(Unsloth) — **24GB 소비자용 GPU 한 장으로는 못 돌린다.** 통합메모리 64GB 이상 Mac이나 멀티 GPU 환경용
- **한국어**: 다국어 학습은 했으나 한국어 품질 평가는 Qwen3 대비 아쉽다는 피드백이 많은 편
- **코딩**: 코드 생성 자체는 무난하나 한국어 지시 → 코드 변환 정확도는 상대적으로 낮다는 평

### Phi-4-mini (Microsoft)

- **파라미터**: 3.8B (dense)
- **라이선스**: MIT
- **특징**: 초경량. 수학/추론 특화 학습으로 크기 대비 성능이 좋다는 것이 기술 보고서의 포지셔닝
- **한국어**: 영어 중심 학습. 한국어는 기본 대화 수준
- **코딩**: 간단한 함수 생성, 코드 자동완성 보조용으로 적합

### DeepSeek V3.2 (DeepSeek)

- **파라미터**: 총 685B (MoE, 활성 약 37B)
- **라이선스**: **MIT** (Hugging Face 모델 저장소 LICENSE 기준)
- **특징**: 추론·코딩 벤치마크 최상위권. 그러나 총 685B라 Q4 양자화 GGUF도 약 400GB급(동급 671B인 DeepSeek-V3의 Q4_K_M이 404GB, Unsloth 기준). **개인 PC 로컬 실행은 사실상 불가 — 공식 API나 클라우드 호스팅으로 쓰는 모델**
- **한국어**: 한국어 품질도 무난하다는 평가가 있으나, 로컬 비교 대상은 아님

## VRAM별 실행 가능 모델 (총 파라미터 적재량 기준, 정정판)

Q4_K_M 양자화 GGUF 파일 크기 기준. 실제로는 KV 캐시·컨텍스트용 여유가 1~수 GB 더 필요하므로, 파일 크기가 VRAM에 딱 맞으면 컨텍스트를 줄이거나 일부 레이어를 CPU로 내려야 한다.

### 단일 소비자용 GPU로 실행 가능

| VRAM | 모델 | Q4 파일 크기 | 적합한 용도 |
|------|------|-------------|------------|
| 8GB | Qwen3-4B | 약 2.5GB | 간단한 챗봇, 요약 |
| 8GB | Phi-4-mini (3.8B) | 약 2.5GB | 코드 자동완성, 수학 |
| 8GB | Qwen3-8B | 약 5GB | 범용 (8GB의 상한선) |
| 12~16GB | Qwen3-14B | 약 9GB | 한국어 챗봇, 번역 |
| 16GB | Qwen3-30B-A3B | 18.6GB | 범용 — 단, 16GB에는 다 안 들어가서 **일부 레이어 CPU 오프로딩 필요**(시스템 RAM 32GB 이상 권장) |
| 24GB | Qwen3-30B-A3B | 18.6GB | 범용 (VRAM에 전부 적재, MoE라 응답 빠른 편) |
| 24GB | Qwen3-32B | 약 20GB | 코딩, 분석, 추론 |

### 고사양·서버급 (참고용)

| 필요 메모리 | 모델 | Q4 파일 크기 | 비고 |
|------------|------|-------------|------|
| 통합메모리 64GB+ (Mac) 또는 멀티 GPU 합산 ~70GB | Llama4-Scout (총 109B) | 65.4GB | 긴 문서 처리. 소비자 GPU 1장으로는 불가 |
| **80GB GPU 2장 이상** 또는 통합메모리 192GB급 | Qwen3-235B-A22B (총 235B) | 134~142GB | 상용 API 대체급. A100/H100 80GB 한 장으로도 부족 |
| 약 400GB — **로컬 사실상 불가** | DeepSeek V3.2 (총 685B) | ~400GB | 공식 API 이용 권장 |

파일 크기 출처: Unsloth GGUF 저장소(Hugging Face) — Llama-4-Scout Q4_K_M 65.4GB, Qwen3-235B-A22B Q4 계열 134~142GB, Qwen3-30B-A3B Q4_K_M 18.6GB, DeepSeek-V3(671B) Q4_K_M 404GB.

생성 속도(tok/s)는 GPU·오프로딩 비율·컨텍스트 길이에 따라 편차가 커서 표에 넣지 않았다. 같은 모델이라도 VRAM에 전부 올라가느냐 CPU로 일부 내려가느냐에 따라 몇 배씩 차이 난다.

|------|---------|------------|-------------|---------|
| Qwen3 | Apache 2.0 | 자유 | 자유 | 제한 없음 |
| Phi-4-mini | MIT | 자유 | 자유 | 제한 없음 |
| DeepSeek V3.2 | MIT | 자유 | 자유 | 라이선스 제한 없음 — 제약은 라이선스가 아니라 685B라는 크기 |

라이선스만 보면 4종 중 3종(Qwen3·Phi-4-mini·DeepSeek)이 자유 라이선스다. 즉 상업적 배포에서 실질적인 선택 기준은 라이선스보다 **"내 하드웨어에 들어가느냐"** — 그 기준에서 Qwen3 라인업이 가장 유리하다.

## 로컬 LLM용 하드웨어 체크리스트

위 표를 하드웨어 관점에서 뒤집으면 이렇게 된다. 각 항목은 위 VRAM 표의 요구량과 매칭한 것이며, 이미 있는 부품으로 충분하다면 새로 살 필요는 없다.

**1. GPU — VRAM 용량이 모델 크기를 결정**

- **16GB (예: RTX 4060 Ti 16GB, RTX 5060 Ti 16GB)**: Qwen3-14B(Q4 약 9GB)를 여유 있게, Qwen3-30B-A3B는 CPU 오프로딩 병행으로. 로컬 LLM 입문용 가성비 구간
- **24GB (예: RTX 3090/4090 중고 포함)**: Qwen3-32B·Qwen3-30B-A3B를 통째로 올릴 수 있는, 단일 카드 기준 현실적 상한

<!-- AFFILIATE:coupang RTX 4060 Ti 16GB 등 GPU 검색 링크 -->

**2. 시스템 RAM — CPU 오프로딩과 모델 로딩의 완충지대**

VRAM보다 큰 모델(예: 16GB GPU + Qwen3-30B-A3B 18.6GB)을 돌리려면 넘치는 레이어가 시스템 RAM으로 간다. 32GB가 실용 최저선, MoE 오프로딩까지 고려하면 64GB가 편하다.

<!-- AFFILIATE:coupang DDR5 64GB 램 링크 -->

**3. SSD — 모델 파일 보관소**

Q4 모델 하나가 2.5~20GB, 여러 모델을 받아 비교하다 보면 수백 GB가 금방 찬다. 로딩 속도 때문에 NVMe SSD 1TB 이상을 권장.

## 실행 환경 세팅 (빠른 시작)

로컬 LLM을 돌리는 가장 간단한 방법은 **Ollama** 또는 **llama.cpp** 기반 도구다.

```
# Ollama 설치 후
ollama pull qwen3:14b
ollama run qwen3:14b

# MoE 모델 (24GB VRAM 또는 RAM 오프로딩 환경)
ollama run qwen3:30b-a3b
```

GUI를 선호한다면 **LM Studio**, **Jan**, **GPT4All** 같은 데스크톱 앱을 쓰면 된다. 모두 GGUF 포맷을 지원하며, Hugging Face에서 양자화 모델을 직접 받을 수 있다. LM Studio는 모델별로 "내 메모리에 들어가는지"를 미리 표시해줘서 입문자에게 특히 편하다.

## 정리

2026년 오픈소스 LLM은 개인 GPU 한 장으로도 실용 수준에 도달했다 — 단, "내 VRAM에 총 파라미터가 들어가는 모델"을 골랐을 때 이야기다. 활성 파라미터 숫자에 속아 DeepSeek V3.2 같은 685B급을 로컬 목표로 잡으면 시간만 버린다. 그 급은 API로 쓰고, 로컬은 Qwen3 라인업(4B~32B, 30B-A3B)에서 VRAM에 맞춰 고르는 것이 현재 가장 합리적인 경로다.

최신 정보 검색, 멀티모달, 초장문 생성은 여전히 클라우드 API가 우위다. 용도에 따라 로컬과 클라우드를 병행하는 것이 현실적 전략이다.
