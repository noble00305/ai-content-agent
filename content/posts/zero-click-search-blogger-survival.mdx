## AI Overview가 바꾼 게임

구글 AI Overview(구 SGE)는 검색 결과 상단에 AI가 생성한 요약을 보여준다. 문제는 이 요약이 블로그 글의 핵심 내용을 그대로 가져다 쓴다는 점이다.

**AI Overview가 답하기 쉬운 콘텐츠** (= 클릭이 줄어드는 유형):
- "X란 무엇인가" 류 정의형
- 단순 목록 나열 ("추천 10선")
- 숫자/날짜/사실 확인형 ("아이폰 17 출시일")
- 기본적인 방법론 ("파이썬 설치 방법")

**AI Overview가 답하기 어려운 콘텐츠** (= 클릭이 유지되는 유형):
- 비교 분석 + 선택 가이드 (맥락 의존적)
- 원본 데이터/벤치마크 (AI가 생성 못함)
- 시각 자료 중심 (스크린샷, 차트, 인포그래픽)
- 최신 변경사항 반영 (AI 학습 데이터 지연)
- 복잡한 조건부 판단 ("내 상황에서 A vs B")

## 콘텐츠 유형별 생존 확률

| 콘텐츠 유형 | 생존 확률 | 이유 |
|------------|----------|------|
| "X란 무엇인가" | 낮음 | AI Overview가 즉답 |
| "추천 N선" (단순 나열) | 낮음 | 리스트를 AI가 요약 가능 |
| "A vs B 비교" (표 + 판단 기준) | 중간~높음 | 맥락/조건에 따라 답이 달라져 클릭 유도 |
| 원본 벤치마크/실험 데이터 | 높음 | AI가 원본 데이터 생성 불가 |
| 단계별 튜토리얼 (스크린샷 포함) | 중간 | 시각 자료 때문에 클릭 발생 |
| 오피니언/분석 칼럼 | 중간 | 신뢰 기반. 브랜드 필요 |
| 실시간 업데이트 콘텐츠 | 높음 | AI 학습 데이터 지연으로 최신 정보 부재 |

## 생존 전략 5가지

### 1. 롱테일 키워드 공략

AI Overview는 검색 볼륨이 높은 일반 쿼리에서 주로 작동한다. 구체적이고 조건부인 롱테일 키워드에서는 AI Overview가 뜨지 않거나, 떠도 정확도가 떨어진다.

- X: "AI 도구 추천"
- O: "프리랜서 디자이너 로고 생성 AI 도구 무료 비교"

### 2. 직접 트래픽 채널 구축

검색 유입에만 의존하면 구글 알고리즘 변경 한 번에 트래픽이 반토막 난다. 검색 외 트래픽 확보가 필수다.

| 채널 | 장점 | 단점 |
|------|------|------|
| 이메일 뉴스레터 | 가장 안정적, 알고리즘 무관 | 구독자 확보에 시간 소요 |
| Threads/X (SNS) | 빠른 확산 | 플랫폼 종속, 알고리즘 변동 |
| 디스코드/오픈채팅방 | 충성 독자 확보 | 운영 리소스 필요 |
| RSS | 기술 커뮤니티에서 여전히 유효 | 일반 유저 사용률 낮음 |

### 3. 콘텐츠 형식 전환

AI가 요약하기 어려운 형식으로 전환한다.

- **비교표 + 선택 플로우차트**: 사용자 조건에 따라 다른 결론
- **인터랙티브 콘텐츠**: 계산기, 퀴즈, 필터 등
- **비디오/스크린캐스트 삽입**: 텍스트만으로는 전달 불가능한 콘텐츠
- **원본 데이터 발행**: 직접 조사/실험한 데이터

### 4. 브랜드 구축

"AI 도구 비교"를 검색하는 사람은 아무 블로그나 클릭한다. 하지만 "이 블로그의 AI 도구 비교"를 찾는 사람은 직접 방문한다. 브랜드 검색(사이트명 + 키워드)이 늘어야 AI Overview에 의존하지 않을 수 있다.

### 5. AI Overview 소스가 되기

역발상이지만, AI Overview에 인용되는 소스가 되면 오히려 권위가 올라간다. 구글은 AI Overview에서 출처 링크를 제공하며, 이 링크의 CTR이 일반 오가닉 결과보다 높다는 보고도 있다.

조건: 명확한 구조 (H2/H3 태그), 정확한 데이터, E-E-A-T 신호

## 이 블로그(AI Content Agent)의 대응

이 블로그 자체도 같은 문제에 직면해 있다. 현재 적용 중인 전략:

- 모든 글에 비교표 + 선택 가이드 포함 → AI Overview로 대체하기 어려운 형식
- 벤치마크 점수, HN 점수 등 구체적 수치 인용 → 원본 데이터 가치
- Threads 자동화 + RSS로 검색 외 유입 경로 확보
- 롱테일 키워드 우선 공략

## 정리

제로클릭 비율 68%는 "블로그가 죽는다"는 뜻이 아니다. "단순 정보 나열형 블로그가 죽는다"는 뜻이다. 직접 봐야 가치 있는 콘텐츠 — 비교 분석, 원본 데이터, 인터랙티브 요소 — 는 오히려 AI Overview 시대에 더 차별화된다.

검색 트래픽이 줄어드는 것은 막을 수 없다. 대신 줄어든 클릭을 가져오는 콘텐츠의 기준이 올라간 것이다. 그 기준에 맞추든, 검색 외 채널을 만들든 — 둘 다 하는 것이 현실적 답이다.
