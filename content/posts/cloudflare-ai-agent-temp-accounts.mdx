## 문제: AI 에이전트에게 누구 계정을 줄 것인가

AI 에이전트가 웹에서 작업하려면 인증이 필요하다. 기존 방식의 문제:

| 방식 | 문제 |
|------|------|
| 개발자 계정 공유 | 보안 위험, 과금 폭탄, 권한 제어 불가 |
| 서비스 계정 생성 | 에이전트마다 수동 설정 필요 |
| API 키 발급 | 키 유출시 전체 계정 위험 |

핵심: **에이전트는 사람이 아닌데, 인프라는 사람 기준으로 설계되어 있다.**

## Cloudflare의 해법

Docker 컨테이너와 같은 철학: "필요할 때 만들고, 쓰고, 버린다."

1. 에이전트 작업 시작 → 임시 계정 자동 생성
2. 필요한 권한만 부여 (최소 권한 원칙)
3. 작업 완료 → 자동 만료
4. 비용은 실사용량만 과금
