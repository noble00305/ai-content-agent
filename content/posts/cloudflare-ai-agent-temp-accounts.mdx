## AI 에이전트는 "계정"이 필요하다 — 그런데 누구 계정으로?

AI 에이전트가 웹에서 작업을 수행하려면 인증이 필요하다. 결제를 하든, API를 호출하든, 데이터를 저장하든. 문제는 이 에이전트가 사람이 아니라는 것이다.

Cloudflare가 이 문제에 대한 답을 내놨다. AI 에이전트 전용 **임시 계정(Temporary Accounts)** 시스템이다. Hacker News에서 169점을 기록하며 개발자 커뮤니티의 뜨거운 관심을 받고 있다.

## 기존 방식의 문제점

AI 에이전트에게 인프라를 제공하는 기존 방식은 크게 세 가지였다:

| 방식 | 문제점 |
|------|--------|
| 개발자 개인 계정 공유 | 보안 위험, 과금 폭탄, 권한 제어 불가 |
| 서비스 계정 생성 | 복잡한 설정, 에이전트마다 수동 생성 필요 |
| API 키만 발급 | 키 유출시 전체 계정 위험, 세분화된 권한 불가 |

핵심 문제는 **"에이전트는 사람이 아닌데, 인프라는 사람 기준으로 설계되어 있다"**는 것이다.

## Cloudflare의 해법: 임시 계정

Cloudflare의 접근법은 단순하다:

1. **에이전트가 작업을 시작할 때** 임시 계정을 자동 생성
2. **필요한 권한만** 부여 (최소 권한 원칙)
3. **작업이 끝나면** 자동 만료
4. **비용은 실제 사용량만** 과금

이건 Docker 컨테이너의 철학과 비슷하다. "필요할 때 만들고, 쓰고, 버린다."

## 한국 개발자에게 의미하는 것

Cloudflare의 이 움직임은 더 큰 트렌드의 일부다: **AI 에이전트가 1급 시민(first-class citizen)이 되는 인프라 시대**.

지금까지 AI 에이전트는 인프라 관점에서 "사람이 쓰는 도구"였다. 사람의 계정으로, 사람의 권한으로, 사람의 과금 체계로 운영됐다.

앞으로는 에이전트 자체가 독립적인 실체로 인프라에 존재하게 된다. 이건 마이크로서비스가 모놀리식을 대체한 것만큼 큰 패러다임 전환이 될 수 있다.
