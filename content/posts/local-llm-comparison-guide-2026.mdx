## 왜 지금 로컬 LLM인가

클라우드 API 비용이 프로젝트 규모에 따라 월 수십~수백 달러까지 올라가는 상황에서, 로컬 LLM은 비용 0원이라는 압도적 장점이 있다. 2026년 기준 오픈소스 모델들의 성능이 GPT-4 수준에 근접하면서, "굳이 API 써야 하나?"라는 질문이 현실적으로 바뀌었다.

개인정보 처리, 오프라인 환경, 커스텀 파인튜닝 등 로컬 실행이 필수인 시나리오도 늘고 있다.

## 모델별 상세 비교

### Qwen3 시리즈 (Alibaba Cloud)

- **파라미터**: 0.6B / 1.7B / 4B / 8B / 14B / 32B / 30B-A3B(MoE) / 235B-A22B(MoE)
- **라이선스**: Apache 2.0 (상업적 사용 완전 자유)
- **특징**: MoE 구조로 실제 활성 파라미터 대비 성능이 높음. 30B-A3B 모델은 3B만 활성화되면서 30B급 지식 보유
- **한국어**: 학습 데이터에 한국어 비중이 높아 자연스러운 한국어 생성 가능
- **코딩**: LiveCodeBench 기준 상위권. 특히 Qwen3-235B-A22B는 Claude Sonnet 4 급

### Llama 4 시리즈 (Meta)

- **파라미터**: Scout (17B-16E, 활성 17B) / Maverick (17B-128E, 활성 17B)
- **라이선스**: Llama Community License (월 7억 MAU 이하 무료, 이상은 별도 협의)
- **특징**: Scout는 1000만 토큰 컨텍스트 윈도우. Maverick은 128개 전문가 중 선택 활성화
- **한국어**: 다국어 학습은 했으나 한국어 품질은 Qwen3보다 떨어짐
- **코딩**: 코드 생성 자체는 괜찮으나, 한국어 지시 → 코드 변환에서 정확도 하락

### Phi-4-mini (Microsoft)

- **파라미터**: 3.8B
- **라이선스**: MIT (완전 오픈)
- **특징**: 초경량. 3.8B임에도 불구하고 7~8B급 성능. 수학/추론 특화
- **한국어**: 영어 중심 학습. 한국어는 기본적인 대화 수준
- **코딩**: 크기 대비 코딩 성능 우수. 간단한 함수 생성/디버깅에 적합

### DeepSeek V3.2 (DeepSeek)

- **파라미터**: 685B (MoE, 활성 37B)
- **라이선스**: DeepSeek License (연구/상업 모두 허용, 일부 제한)
- **특징**: 전체 파라미터 수는 685B로 거대하나 MoE 덕분에 활성 37B만 사용. 추론 능력 최상위
- **한국어**: 중국어 기반이라 한자권 언어 이해도가 높고, 한국어도 상대적으로 양호
- **코딩**: 코딩 벤치마크 최상위권. 복잡한 멀티파일 리팩토링에도 강함

## 실행 환경 세팅 (빠른 시작)

로컬 LLM을 돌리는 가장 간단한 방법은 **Ollama** 또는 **llama.cpp** 기반 도구를 사용하는 것이다.

```
# Ollama 설치 후
ollama pull qwen3:14b
ollama run qwen3:14b

# 또는 특정 양자화 버전
ollama pull qwen3:30b-a3b-q4_K_M
```

GUI를 선호한다면 **LM Studio**, **Jan**, **GPT4All** 등의 데스크톱 앱을 사용할 수 있다. 모두 GGUF 포맷 모델을 지원하며, Hugging Face에서 양자화 모델을 직접 다운로드할 수 있다.

## 정리

2026년 오픈소스 LLM 생태계는 "쓸 만한 수준"을 넘어 "상용 API를 대체할 수 있는 수준"에 도달했다. 특히 Qwen3의 Apache 2.0 라이선스 + MoE 효율 조합은 개인 개발자에게 현실적인 선택지다.

다만 모든 작업에 로컬 LLM이 최선은 아니다. 최신 정보가 필요한 실시간 검색, 멀티모달 처리, 초장문 생성 등에서는 여전히 클라우드 API가 우위에 있다. 용도에 맞게 로컬과 클라우드를 병행하는 것이 현실적 전략이다.
