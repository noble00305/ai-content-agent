## 무슨 일인가

칭화대 기반 GLM 팀이 공개한 벤치마크에서, MIT 라이선스 오픈소스 모델 GLM-5.2가 OpenAI의 GPT-5.5보다 환각 발생률이 약 3분의 1 수준으로 나타났다. HN에서 889점을 기록하며 큰 반향을 일으켰다.

환각(hallucination)은 AI가 **없는 정보를 있는 것처럼 생성**하는 현상이다.

## 왜 환각이 중요한가

| 분야 | 환각 사고 사례 | 리스크 |
|------|--------------|--------|
| 법률 | 존재하지 않는 판례 인용 | 소송 패소, 변호사 징계 |
| 의료 | 약물 상호작용 허위 설명 | 환자 안전 위협 |
| 금융 | 가짜 통계 기반 분석 | 투자 손실 |
| 코드 | 없는 API 메서드 추천 | 런타임 에러, 보안 취약점 |
| 학술 | 가짜 논문/저자 생성 | 연구 신뢰도 훼손 |

핵심: 환각의 문제는 **틀린다는 것**이 아니라, **맞는 것처럼 보인다**는 것이다.

## 숫자만 보면 안 되는 이유 3가지

**1. 벤치마크 ≠ 실무**

환각 벤치마크는 특정 데이터셋으로 측정한다. 실제 업무에서의 환각 패턴은 다를 수 있다. "우리 도메인"에서 직접 테스트하는 게 유일한 답이다.

**2. 환각을 줄이는 대신 잃는 것**

GLM-5.2가 환각이 적은 이유 중 하나는 "모르면 안 답한다"는 설계 때문이다. 이건 정확도에서는 장점이지만, 탐색적 질문이나 창의적 작업에서는 단점이 된다.

**3. 환각은 프롬프트로 줄일 수 있다**

"확실하지 않으면 '모르겠다'고 답하라", "출처를 함께 제시하라" 같은 프롬프트 기법으로 어떤 모델이든 환각률을 상당히 줄일 수 있다.

## 오픈소스 AI가 의미하는 것

GLM-5.2의 MIT 라이선스가 주는 실질적 이점:

- **비용**: API 과금 없음. GPU 서버 비용만 (월 ~$2,000~3,000)
- **데이터 주권**: 고객 데이터가 외부 서버로 나가지 않음
- **커스터마이징**: 자사 도메인 데이터로 파인튜닝 가능
- **장애 독립**: OpenAI 서버 장애와 무관하게 운영

단, **자체 호스팅 = 인프라 운영 부담**이므로 팀에 ML Ops 역량이 있어야 한다.
