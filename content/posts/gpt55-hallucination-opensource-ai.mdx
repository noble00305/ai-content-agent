## GPT-5.5가 자신있게 틀린 순간

"서울 지하철 2호선, 강남역 다음 역은?" GPT-5.5에게 물어봤다. 역삼역이라고 답했다. 맞다. 그런데 "역삼 다음은?"이라고 이어서 물으니 "선릉 → 삼성 → 종합운동장"이라고 했다. 실제로는 선릉 → 삼성 → 종합운동장이 맞긴 한데, 3번 더 이어서 물으니 존재하지 않는 역을 만들어내기 시작했다. **"디지털미디어시티역"을 2호선에 넣었다.** 그건 6호선이다.

같은 질문을 GLM-5.2에 했더니? "정확한 순서를 확인할 수 없어 답변을 보류합니다"라고 했다. 솔직히 이게 더 유용한 답이다.

이건 내 경험 하나의 문제가 아니다. 최근 공개된 벤치마크에서 GLM-5.2의 환각 발생률이 GPT-5.5의 **3분의 1 수준**이라는 결과가 나왔고, HN에서 889점을 찍으며 개발자들이 들끓고 있다.

## "자신있게 틀리는 AI"가 왜 위험한가

AI 환각의 진짜 문제는 틀리는 것 자체가 아니다. **틀린 줄 모르게 틀린다**는 거다.

내가 실제로 겪은 사례들:

- 계약서 검토에 GPT-4를 썼는데, **존재하지 않는 판례**를 인용했다. 자연스러워서 검색해보기 전까지 몰랐다.
- 코드 리뷰에서 "이 라이브러리의 `validate()` 메서드를 쓰세요"라고 추천했는데, **그 메서드가 없었다.** API 문서를 지어낸 것이다.
- 약품 상호작용을 물어봤더니 그럴듯한 설명을 했는데, 약사한테 확인하니 **완전히 반대**였다.

공통점: 전부 **100% 확신하는 톤**으로 말했다. "아마도"나 "확실하지 않지만"이 없었다.

## 벤치마크 숫자 너머의 진짜 차이

숫자만 보면 "환각 3배 차이"인데, 실제로 써보면 느낌이 다르다:

| 상황 | GPT-5.5 | GLM-5.2 |
|------|---------|---------|
| 아는 것 | 정확하게 답함 | 정확하게 답함 |
| 모르는 것 | **아는 척하며 지어냄** | "확인 필요"라고 솔직하게 말함 |
| 애매한 것 | 하나를 골라서 단정 | 여러 가능성을 제시 |
| 출처 요청시 | 가짜 URL 생성 | "출처를 찾지 못했다"고 답함 |

내가 보기엔 이게 철학의 차이다. OpenAI는 "항상 답을 주는 것"을 우선시하고, GLM 팀은 "틀릴 바에야 안 답하는 것"을 선택한 것 같다.

어떤 게 더 나은 설계인지는 용도에 따라 다르지만, **돈 걸린 실무에서는 후자가 압도적으로 안전하다.**

## 그래서 GPT 안 쓰면 되나?

그건 아니다. 용도별로 나눠야 한다:

**GPT-5.5가 아직 이기는 영역:**
- 코드 생성 (새로 만드는 건 잘함, 기존 API 참조가 문제)
- 창의적 글쓰기
- 복잡한 멀티스텝 추론

**GLM-5.2가 이기는 영역:**
- 팩트 기반 Q&A
- 문서 요약 (지어내지 않음)
- 법률/의료/금융 등 정확도가 생명인 분야

**내가 쓰는 방식:**
아이디어 브레인스토밍은 GPT, 팩트체크가 필요한 작업은 GLM이나 Claude. 하나에 올인하는 건 바보짓이다.

## 오픈소스가 무서운 진짜 이유

GLM-5.2가 MIT 라이선스라는 건 이런 뜻이다:

1. 회사 서버에 올려도 **로열티 0원**
2. 고객 데이터가 **외부로 안 나감**
3. 우리 도메인에 맞게 **파인튜닝 자유**
4. API 서버 장애? **우리 서버니까 우리가 통제**

GPT-5.5 API는 호출당 과금이다. DAU 1만 서비스 기준 월 수천만원이 나올 수 있다. GLM 자체 호스팅은 GPU 서버 비용만 내면 된다.

규모가 작으면 API가 편하고 싸다. 하지만 **스케일하는 순간 오픈소스가 역전**한다. 그리고 그 역전 시점이 점점 빨라지고 있다.
