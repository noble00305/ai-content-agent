## 자체 호스팅 vs API — 언제 갈아타야 하나

오픈소스 모델을 직접 돌리는 게 언제 이득인지 계산해보자.

**A100 GPU 서버 (클라우드) 월 비용: 약 $2,000~3,000**

이 서버로 Llama 4 70B를 돌리면:
- 초당 약 30토큰 생성
- 하루 약 260만 출력 토큰
- 월 약 7,800만 출력 토큰

**손익분기점:**

```
API 비용 > GPU 비용이 되는 지점:

GPT-4o mini 기준:
7,800만 토큰 x $0.60/M = $46.8
→ GPU 비용 $2,500 >>> API 비용 $46.8
→ 자체 호스팅이 손해

GPT-4o 기준:
7,800만 토큰 x $10/M = $780
→ 아직 GPU가 비쌈

월 2.5억 출력 토큰 이상 → 자체 호스팅이 이득 (GPT-4o 급 기준)
```

**의사결정 프레임워크:**

| 월 토큰 사용량 | 추천 |
|:---:|------|
| ~1억 이하 | API 호출 (경량 모델) |
| 1~5억 | API 호출 + 캐싱 최적화 |
| 5억~20억 | 하이브리드 (핵심은 API, 나머지 자체호스팅) |
| 20억 이상 | 자체 호스팅 본격 검토 |

## 비용 줄이는 실전 팁

1. **프롬프트 캐싱** — 같은 시스템 프롬프트를 반복 전송하지 말 것. Anthropic, OpenAI 모두 프롬프트 캐싱 지원.
2. **모델 라우팅** — 쉬운 질문은 Haiku, 어려운 질문만 Sonnet/GPT-4o. 비용 50~70% 절감 가능.
3. **출력 길이 제한** — max_tokens를 타이트하게. 불필요한 토큰이 가장 큰 낭비.
4. **배치 API** — 실시간이 아닌 작업은 배치로. OpenAI 배치 API는 50% 할인.
