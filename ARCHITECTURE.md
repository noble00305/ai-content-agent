# AI Autonomous Content Agent - System Architecture

## 철학
이 시스템은 "자동화 스크립트"가 아니라 "자율 에이전트"다.
차이점: 스크립트는 정해진 대로 실행한다. 에이전트는 상황을 판단하고 스스로 결정한다.

---

## 1. 시스템 전체 구조

```
                    [Human]
                      |
                  승인/거부/피드백
                      |
                      v
+=====================================================+
|                 AGENT BRAIN (핵심)                    |
|  "모든 판단의 중심. 다른 모듈은 손발이다"              |
|                                                     |
|  - 오늘 뭘 할지 스스로 결정                           |
|  - 성과 데이터 보고 전략 수정                         |
|  - 리스크 판단해서 사람한테 물어볼지 결정              |
|                                                     |
|  State: 현재 전략, 목표, 학습된 패턴                  |
+=====================================================+
        |           |           |           |
        v           v           v           v
  [Eyes 모듈]  [Hands 모듈]  [Memory]   [Guardian]
   정보 수집    실행/발행     기억/학습   통제/승인
```

---

## 2. 모듈 상세 설계

### 2.1 BRAIN (두뇌) - 의사결정 엔진

역할: 모든 판단의 중심. 매 사이클마다 "지금 뭘 해야 하는가"를 결정.

```
입력:
  - Eyes가 수집한 트렌드 데이터
  - Memory에서 가져온 과거 성과
  - Guardian의 제약 조건
  - Human의 피드백/승인 결과

처리 (Claude API):
  1. 현재 상황 분석 (트렌드 + 성과 + 리소스)
  2. 오늘의 행동 계획 수립
     - 어떤 주제로 글을 쓸지
     - 어떤 채널에 발행할지
     - 기존 글 중 뭘 개선할지
     - 새로운 실험을 할지
  3. 리스크 판단
     - 이 결정이 승인 필요한가?
     - 자동 실행해도 되는가?

출력:
  - Action Plan (오늘의 할 일 리스트)
  - 각 Action의 리스크 등급
```

#### Brain의 자율 판단 예시:
```
상황: "AI 에이전트" 키워드 글이 최근 3편 연속 높은 조회수
판단: 이 주제 시리즈물로 확장하자
행동: 관련 세부 주제 5개 도출 -> 이번 주 집중 발행

상황: 특정 글의 이탈률이 80% 이상
판단: 도입부가 약하다
행동: 도입부 A/B 테스트 버전 생성 -> 교체

상황: 새로운 AI 모델 출시 뉴스 (긴급 트렌드)
판단: 속보성 콘텐츠 즉시 발행 필요
행동: 기존 계획 밀고 속보 글 우선 작성 -> 자동 발행
```

### 2.2 EYES (눈) - 정보 수집 모듈

역할: 외부 세계를 관찰하고 Brain에 전달

```
[트렌드 수집기]
  - Hacker News API -> 인기 포스트
  - Reddit r/artificial, r/technology -> 핫 토픽
  - X(Twitter) -> AI 관련 바이럴 트윗
  - Product Hunt -> 신규 AI 도구
  - arXiv -> 주요 AI 논문
  - Google Trends -> 검색량 변화
  - 경쟁 블로그 RSS -> 경쟁사 동향

[성과 수집기]
  - Google Analytics API -> PV, 체류시간, 이탈률
  - Google AdSense API -> 광고 수익, CPM, CPC
  - Search Console API -> 검색 노출, CTR, 키워드 순위
  - SNS API -> 좋아요, 공유, 댓글 수

[수집 주기]
  - 트렌드: 매 6시간
  - 성과: 매일 1회 (오전 9시)
  - 긴급 트렌드: 실시간 웹훅 (선택)
```

---

### 2.3 HANDS (손) - 실행 모듈

역할: Brain의 결정을 실제로 수행

```
[콘텐츠 생성기]
  입력: Brain의 Action Plan (주제, 톤, 길이, 키워드)
  처리:
    1. 주제 리서치 (웹 검색으로 팩트 수집)
    2. 글 구조 설계 (SEO 최적화 구조)
    3. 초안 작성 (Claude API)
    4. 자체 검수 (팩트체크, 가독성, SEO 점수)
    5. 썸네일 텍스트 생성
    6. 메타 디스크립션 생성
  출력: 발행 준비된 콘텐츠

[발행기]
  - 블로그: MDX 파일 생성 -> Git push -> Vercel 자동 배포
  - 인스타: 카드뉴스 텍스트 -> 이미지 생성 API -> 자동 포스팅
  - X(Twitter): 요약 스레드 자동 작성 -> 발행
  - 유튜브 쇼츠: 스크립트 생성 -> TTS -> 자동 업로드 (Phase 2)

[최적화기]
  - 기존 글 제목 A/B 테스트
  - 메타 디스크립션 개선
  - 내부 링크 자동 추가
  - 오래된 글 업데이트 (정보 최신화)
```

---

### 2.4 MEMORY (기억) - 상태/학습 저장소

역할: 에이전트의 장기 기억. 경험에서 배운다.

```
[전략 메모리] strategy.json
  - 현재 콘텐츠 전략 (집중 키워드, 타겟 독자)
  - 채널별 발행 전략
  - 수익 목표 및 현재 달성률

[성과 메모리] performance.db
  - 글별 성과 기록 (PV, 체류시간, 수익)
  - 주제별 평균 성과
  - 시간대별 발행 효과
  - 제목 패턴별 CTR

[학습 메모리] learned_patterns.json
  - "~하는 제목이 CTR 2배 높음"
  - "화요일 오전 발행이 가장 효과적"
  - "시리즈물이 단발성 글보다 체류시간 40% 높음"
  - "코드 예시 포함 글이 공유 3배 많음"

[실패 메모리] failures.json
  - 실패한 실험과 이유
  - 피해야 할 패턴
  - 사람이 거부한 결정과 이유
```

---

### 2.5 GUARDIAN (수호자) - 통제/승인 시스템

역할: 에이전트의 행동을 통제하고, 필요시 사람에게 판단을 넘김

```
[리스크 엔진]
  모든 Action에 대해 리스크 점수 산정 (0~100)

  자동 실행 (0~30):
    - 기존 전략 내 콘텐츠 발행
    - SEO 메타 수정
    - 내부 링크 추가
    - 성과 리포트 생성

  알림 후 실행 (31~60):
    - 새로운 주제 카테고리 시도
    - 발행 시간 변경
    - 콘텐츠 톤 변경 실험
    - 기존 글 대폭 수정

  승인 필수 (61~100):
    - 새 채널 진출 (유튜브 등)
    - 광고 계약/제휴 제안 수락
    - 사이트 구조 변경
    - 브랜드 방향 전환
    - 비용 발생하는 결정
    - 민감한 주제 (정치, 종교 등)

[승인 큐]
  - 승인 필요 액션 -> 대시보드에 등록
  - 알림 발송 (카카오톡/슬랙/이메일)
  - 타임아웃: 24시간 미응답시 리마인더
  - 72시간 미응답시 자동 보류 + 대체 행동

[감사 로그]
  - 모든 결정과 실행 기록
  - 왜 그 결정을 했는지 근거 저장
  - 사람이 언제든 추적 가능
```

## 3. 일일 운영 사이클

```
[매일 오전 6:00] Eyes 가동
  -> 밤사이 트렌드 수집
  -> 전일 성과 데이터 수집

[매일 오전 7:00] Brain 가동
  -> 수집 데이터 + Memory 분석
  -> 오늘의 Action Plan 수립
  -> 리스크 판단 -> Guardian 검증

[매일 오전 8:00~18:00] Hands 실행
  -> 콘텐츠 생성 (2~5편/일)
  -> 최적 시간에 자동 발행
  -> 기존 글 최적화 작업

[매일 오후 21:00] 성과 분석
  -> 오늘 발행 콘텐츠 초기 반응 체크
  -> Memory 업데이트
  -> 내일 전략에 반영

[매주 월요일] 주간 회고
  -> Brain이 주간 성과 종합 분석
  -> 전략 수정 여부 판단
  -> 주간 리포트 -> 사람에게 발송
```

## 5. 기술 스택

```
인프라:
  - Vercel (블로그 호스팅 + API Routes)
  - Supabase (DB + Auth + Realtime)
  - GitHub Actions (스케줄 실행)
  - Upstash Redis (큐, 캐시, 레이트리밋)

AI:
  - Claude API (Brain + 콘텐츠 생성)
  - 프롬프트 체인 (리서치 -> 구조 -> 작성 -> 검수)

프론트엔드:
  - Next.js 15 (블로그 + 관리자 대시보드)
  - Tailwind CSS
  - shadcn/ui

데이터:
  - Supabase PostgreSQL (성과, 로그, 승인큐)
  - JSON 파일 (전략, 학습 패턴) -> Git 관리

외부 연동:
  - Google Analytics Data API
  - Google AdSense API
  - Google Search Console API
  - X(Twitter) API
  - Instagram Graph API
```

## 6. 성장 단계

### Phase 1: 기반 구축 (Week 1~2)
- 블로그 사이트 구축 + 배포
- Brain 기본 로직 (트렌드 -> 글 생성 -> 발행)
- 승인 대시보드 기본
- 매일 2편 자동 발행 시작

### Phase 2: 자율성 강화 (Week 3~4)
- Memory 시스템 구축 (성과 기록 + 학습)
- Brain 고도화 (데이터 기반 판단)
- SEO 최적화 자동화
- SNS 자동 발행 연동

### Phase 3: 확장 (Month 2~3)
- 숏폼 콘텐츠 자동 생성
- A/B 테스트 자동화
- 수익 최적화 (광고 배치, 제휴)
- 피드백 루프 완성

### Phase 4: 완전 자율 (Month 3+)
- Brain이 스스로 전략 수정
- 새로운 콘텐츠 포맷 실험
- 채널 확장 자율 제안
- 사람은 주간 리포트만 확인
